Regarding the current stage of AIGC and vibe coding, I feel more and more that it is not a single button that can directly move an idea from the inside of our head to a finished thing. It is more like a table with many tools on it. Some of them are good at understanding the vague part of an idea, some of them are good at writing code patiently, some of them are good at making images, and some of them are only suitable for helping us check whether a sentence or a structure is strange.

At the beginning, I also had a very natural expectation that maybe one model, one prompt, and one generation could finish everything. This expectation is very attractive, because it makes creation look like a shortcut. We only need to describe what we want, wait for a few seconds, and then receive a complete result. But after using these tools for more real work, I gradually feel that this is not always the best way to work with AI.

The reason is simple. Different models have different personalities in practice. Some models are better at long context and can keep a large project in mind for a longer time. Some models are more sensitive to visual taste and can help us find a more interesting direction in an image or interface. Some models are fast and cheap, so they are suitable for doing small transformations again and again. Some models are more careful in coding, but may not be the most imaginative when we ask them to create a concept from nothing.

So instead of asking one model to become a perfect creator, I prefer to treat them as different parts of a creative workflow. For example, we can use one model to collect and organize the first rough idea, use another one to expand the structure, then use a coding model to turn the structure into a real project, and use a visual model to explore assets or atmosphere. After that, we still need to come back as a human, read the result, delete the unnecessary part, adjust the detail, and decide what should stay.

This sounds less magical, but I think it is actually closer to creation.

In vibe coding, this feeling is even stronger. A model can generate a page, a component, or even a whole small application very quickly. However, if we only keep saying "make it better" and wait for the next version, the result can easily become bigger but not clearer. The AI may add more sections, more animations, more options, and more explanations, but the product itself may not become more precise. Sometimes the better move is very manual: remove one button, rewrite one sentence, change a color by ourselves, or tell the model that this part should not be expanded.

I think this manual part is not a failure of AI. It is the part where our taste still exists.

For a long time, digital tools have always been like this. A camera does not remove the need for composition. A code editor does not remove the need for understanding a system. A design tool does not remove the need for deciding what kind of feeling a page should have. In the same way, AIGC does not remove the need for editing. It only changes where the editing happens. We are no longer only editing pixels or code line by line. We are also editing the path of generation itself.

This is why I think combining different models from different companies is a very practical direction. It is not because we need to chase every new model name, or because a complicated workflow looks more professional. It is because different tools can give us different kinds of friction and help. A strong reasoning model may help us find the hidden problem in an idea. A fast model may help us produce many drafts. A model with good coding ability may help us safely change a project. An image model may help us see something we could not describe very clearly in words.

When they are composed together, the workflow becomes more like a studio than a slot machine. We are not pulling once and hoping for luck. We are moving between materials, versions, and decisions. We can let one model produce, let another model criticize, let another model implement, and then let our own hands do the final small but important work.

Of course, this also means we should not use AI with too much obedience. Sometimes the output looks complete, but it is only complete on the surface. The structure may be loose, the visual direction may be generic, or the code may work only in the happy path. If we accept it too early, the work will carry the smell of a first draft. But if we use it as a first draft, or a second brain, or a temporary assistant with a very strong but uneven ability, it becomes much easier to use.

I like the idea that the current stage of AIGC is not about replacing the creator with one perfect machine. It is about giving the creator a larger set of hands. Some hands are quick, some hands are careful, some hands are strange, and some hands need to be stopped before they make the page too noisy. The important thing is still the person who chooses the order, keeps the intention, and knows when to stop.

Maybe one day the tools will become much more stable, and the distance between idea and result will become shorter. But at this moment, I still believe that the better way is to compose different models, keep our own manual work inside the process, and not be too eager to make everything one-step and automatic.

Because creation is not only about getting an output. It is also about staying close enough to the thing we are making, so that it can still feel like it belongs to us.
